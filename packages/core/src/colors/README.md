# Colors

В этой папке хранятся все **цветовые палитры** и отдельные цвета для нашего проекта.  
Цель — иметь централизованный источник цветов, чтобы UI был консистентным и легко поддерживаемым.

---

## Структура папки
```txt
src/colors/
├── brand.ts # Основные брендовые цвета
├── accents.ts # Акцентные цвета (success, warning, error и др.)
├── neutrals.ts # Базовые серо-черные оттенки и белый цвет
└── index.ts # Рекурсивный экспорт всех цветов
```
---

## Общие правила и договорённости

1. **Нейминг констант**
   - Палитры цветов всегда в формате `COLORS_NAME_PALETTE`:

     ```ts
     export const BLUE_COLORS_PALETTE: MantineColorsTuple = [...];
     ```

   - Отдельные однотонные цвета — `COLOR_NAME_COLOR`:

     ```ts
     export const WHITE_COLOR = colorsTuple('#ffffff');
     ```

   - Используем **MantineColorsTuple** или `colorsTuple` из `@mantine/core` для генерации оттенков.

2. **Палитры**
   - Палитра — массив из 10 оттенков (`MantineColorsTuple`), где каждый индекс имеет смысл:
     - `[0]` — самый светлый оттенок
     - `[9]` — самый тёмный оттенок
   - Комментарии рядом с HEX-кодами описывают назначение.

3. **Категории**
   - **brand** — брендовые цвета, фирменные.
   - **accents** — яркие или статусные цвета (hover, danger, success).
   - **neutrals** — базовые цвета интерфейса: серо-черные оттенки для текста, фонов, бордеров, а также белый цвет.

4. **Документация**
   - Каждая палитра или цвет сопровождается JSDoc на русском с кратким описанием:
     - Назначение цвета
     - Индекс в массиве
     - Контекст использования (фон, текст, бордер, hover и т.д.)

## Пример использования

```ts
import { BLUE_COLORS_PALETTE, WHITE_COLOR, BLACK_COLORS_PALETTE } from './colors';

// Использование в компоненте
<Box bg={BLUE_COLORS_PALETTE[0]} color={BLACK_COLORS_PALETTE[6]}>
  Привет, мир!
</Box>
```
